# Contributing

Thank you for your interest in contributing to T.I.M.

T.I.M is currently in active development, and the language is still evolving. Contributions are welcome, but please read the guidelines below before opening an issue or pull request.

---

## Before Contributing

If you plan to make significant changes, please open an issue first so we can discuss the proposal before implementation.

This helps avoid duplicated work and ensures the change aligns with the project's direction.

---

## Reporting Bugs

When reporting a bug, please include:

- The `.tim` source file or a minimal example.
- The expected behavior.
- The actual behavior.
- Any compiler output or error messages.
- The operating system and Rust version, if relevant.

---

## Feature Requests

Feature requests are welcome.

Please explain:

- The problem you're trying to solve.
- Why existing syntax is insufficient.
- How your proposal fits into the language.

Keep in mind that T.I.M intentionally favors a small, deterministic language over feature abundance.

---

## Pull Requests

Before submitting a pull request, ensure that:

- The project builds successfully.
- Existing functionality continues to work.
- New syntax is documented.
- New behavior includes appropriate tests where applicable.
- Code follows the existing project style.

Small, focused pull requests are preferred over large changes.

---

## Design Philosophy

T.I.M follows a few core principles:

- Deterministic parsing.
- Explicit language rules.
- Readable source documents.
- Minimal syntax.
- Compiler-friendly design.

Every new feature should support these principles.

If a feature increases language complexity without providing significant value, it is unlikely to be accepted.

---

## Development Status

The language is still evolving.

Syntax may change between releases until a stable specification is published.

---

## Code of Conduct

Please be respectful and constructive when participating in discussions.

We welcome thoughtful feedback, bug reports, documentation improvements, and implementation contributions.
